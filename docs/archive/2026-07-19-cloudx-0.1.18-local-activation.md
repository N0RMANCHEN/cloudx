Date: 2026-07-19

## Decision And Boundary

The operator explicitly approved every remaining independent Roadmap confirmation while requiring local communication continuity. Cloud `0.1.18` had already passed its separate transaction. The local transaction therefore supplied the distinct gate:

```text
INSTALL CLOUDX LOCAL 0.1.18
```

This authorized exact signed local staging, selector/entrypoint activation, and shell-source reconciliation only. It did not authorize native-profile reseeding, CPA restart or policy activation, watcher activation, credential/archive mutation, cloud mutation, Phi mutation, or legacy retirement.

The operator acceptance script then referenced `sys.platform` without importing `sys`. This script defect triggered fail-closed recovery. The prebuilt signed recovery command restored `current=0.1.17`, retained staged `0.1.18`, restored the snapshotted shell files, and passed CPA health plus a real official-Codex-through-CPA recovery canary. No CPA process restarted.

Read-only inspection confirmed the restored state and fresh-shell mode resolution. The corrected independent retry revalidated the immutable artifact, reported stage `already-staged`, activated signed `0.1.18`, and completed every acceptance assertion.
