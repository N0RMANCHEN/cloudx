# Cloudx 0.1.8 Activation And CPA Health Cutover

## Authorized Scope

The operator approved cloud-first and local-second activation of signed Cloudx `0.1.8`, without reinstalling the shell hook or seeding a native profile. The same maintenance action authorized backup and installation of only the signed `cloudx-cpa-health` service/timer, one systemd daemon reload, and restart of that timer only.

CLIProxyAPI, the old importer, Phi, the quota monitor, legacy health, Tailscale, mihomo, SSH, the local CPA, and the legacy `18317` listener were outside the mutation scope.

The mutable codex-plus checkout is no longer the execution source for production CPA health. `/opt/codex-gateway/codexx_app` remains an explicit compatibility dependency for quota probing and reversible quarantine, so the runtime package and old importer are not yet retirement candidates.
