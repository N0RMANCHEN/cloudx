## Immutable Installer Risk

Signed `0.1.18` was published before the current native-config drift existed. Its immutable canonical `./install local` path always passes `--seed-native-from soul0` after selector activation. An isolated synthetic-home execution proved that this path overwrites an existing complete native config with the seed config. It is therefore not accepted for this workstation because installation must preserve current communication and native workspace state.

Repository development `0.1.19` corrects future installers: a complete native auth/config pair is preserved byte-for-byte; both absent files may be initialized only from a valid regular named-account source; partial, symlinked, non-regular, or invalid-source state fails before backup, fetch, stage, apply, selector, shell, or profile mutation.

That source correction does not and cannot rewrite immutable signed `0.1.18`. The real `0.1.18` maintenance action must use the signed preserving path below.
