## Automatic CPA Update Matching

`cloudx.local-cpa-capabilities.v1` makes upstream CPA replacement self-detecting on every import preview and write:

1. Cloudx safely resolves and hashes the currently selected external CPA executable.
2. The exact digest and required capability must match the mode-`0600` sidecar.
3. The literal loopback `/healthz` service must advertise the same versioned capability.
4. A stale sidecar, changed binary, missing live advertisement, symlink substitution, or non-loopback probe fails before credential mutation with the sanitized external-capability error.
5. A later compatible CPA build is accepted automatically once its exact sidecar and live runtime match; no Cloudx code change is required for the same versioned capability.

This deliberately automates detection and compatibility matching, not external service ownership. Cloudx does not silently download, patch, activate, or restart CPA when upstream changes. Candidate build/staging and operator activation remain separate, rollback-backed actions.
